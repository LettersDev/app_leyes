# INSTRUCCIONES PARA ACTUALIZAR FIREBASE

Tienes dos opciones para subir las leyes corregidas a tu base de datos. Ambas actualizarán el contenido existente en Firestore.

---

## Opción 1: Usando Python (Recomendado para archivos específicos)
Esta opción es ideal si solo quieres subir una ley específica o si prefieres el entorno Python.

1. **Instalar dependencias (solo la primera vez):**
   ```powershell
   pip install firebase-admin
   ```

2. **Subir una ley específica (ej: Código Civil):**
   ```powershell
   python scripts/upload_laws.py --file data/codigo_civil_full.json
   ```

3. **Subir TODAS las leyes de la carpeta data/:**
   ```powershell
   python scripts/upload_laws.py --all
   ```

---

## Opción 2: Usando Node.js (seedDatabase.js)
Este script ya lo tenías en tu proyecto. Sí, hace lo mismo y **SÍ actualiza los datos**. 

**Ventajas de seedDatabase.js:**
- Detecta si hubo cambios reales antes de subir (usa un hash). Si la ley no ha cambiado, se la salta para ahorrar tiempo/cuota.
- Está configurado para usar el SDK de Web, que es más resistente a bloqueos de red.

**Cómo ejecutarlo:**
Desde la raíz del proyecto:
```powershell
node scripts/seedDatabase.js
```

---

### ⚠️ NOTA IMPORTANTE SOBRE EL CÓDIGO CIVIL
Como hemos corregido el Artículo 185 y otros fragmentos, CUALQUIERA de los dos scripts detectará el cambio y actualizará Firestore con la versión corregida. 

**Mi recomendación:** Ejecuta `node scripts/seedDatabase.js` para sincronizar todo de forma eficiente, o usa el de Python si quieres tener más control sobre qué archivo subir.
